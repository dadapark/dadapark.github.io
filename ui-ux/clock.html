Overview

A clock application is one of the most commonly used IOS apps on a daily basis, and there were certain problem spaces discovered during user interviews.

The initial outcome was to obtain solutions to support problems that users currently face, and provide better experiences to users while they complete their tasks in the app.

Challenge

  • Users cannot tap on the alarm they want to edit. It is considered as an unnecessary extra step, waste of time.
  • Users finds it hard to set the time when they edit/add alarm.
  • Users do not know how the snooze feature works.
Users want to use their clock application to set the alarm in a faster and intuitive way

90%

of users want to tap on what they want to edit, rather than tapping on 'edit' button

The traditional way is still there, but an area that users can tap has been added with an indicator on the right edge of the screen.
Only one step needed for getting into the edit screen.

More accessible & visible interface

Currently, the app offers two different methods of setting time. And the interface impacts both types of users; Scroller & Typer. The goal was to provide a bigger room for scrolling, and also more visible information for all users including those who type.

In facts, the survey result showed that both types of users thought highly of the design change.

Useful Snooze

From the data collected from user interviews and surveys, it appeared that most of the users are not aware of how the current snooze feature works. The interval is set as 9 minutes as default, which none of the users knows.

The goal was to make the snooze feature more useful by showing users the current interval and having space where they can set their desired interval if needed.

Being my first UX case study, I incorporated lessons learned from the UX course and previous experiences as designer. However, I still found myself being unprepared for a lot of the challenges I faced while researching and problem-solving. Defining a specific user group for the project was the biggest challenge for me, and I started working with very broad types of users. I believe I could overcome this by allocating more time for user research. In future projects, I hope to further develop my research skills and be able to understand and empathize with users better. Overall, I thoroughly enjoyed working on this project.
